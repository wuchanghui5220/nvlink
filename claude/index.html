📖 简介

Claude Code 是 Anthropic 推出的一个命令行工具，目前处于研究预览阶段。它允许开发者直接在终端中与 Claude 进行交互，帮助处理各种编程任务。

✨ 主要特性：
  • 直接在终端中与 Claude 交互
  • 支持文件读取和修改
  • 会话管理和恢复
  • 管道输入处理
  • 多种计费方式

⚙️ 系统要求

💻 操作系统

macOS / Linux / Windows

🔧 运行环境

Node.js 和 npm

🌐 网络连接

可能需要代理（中国大陆用户）

🔑 账户要求

Claude Pro 订阅或 Anthropic API 密钥

🚀 安装步骤

1. 安装 Claude Code

使用 npm 全局安装：

npm install -g @anthropic-ai/claude-code
⚠️ 如果遇到权限错误（EACCES），可以选择以下解决方案：

方案1：使用 sudo（最简单）

sudo npm install -g @anthropic-ai/claude-code

方案2：修改 npm 全局目录权限（推荐）

# 创建全局目录 mkdir ~/.npm-global # 配置 npm 使用新目录 npm config set prefix '~/.npm-global' # 添加到环境变量 echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc source ~/.zshrc # 现在可以无需 sudo 安装 npm install -g @anthropic-ai/claude-code

方案3：使用 npx（无需全局安装）

npx @anthropic-ai/claude-code

2. 配置网络代理（如需要）

如果您在需要代理的网络环境中（如中国大陆），需要配置终端代理：

确认 Clash 代理端口

  1. 打开 Clash Verge
  2. 查看设置中的 HTTP 代理端口（通常是 7897）

设置代理环境变量

临时设置（仅当前会话）：

export HTTP_PROXY=http://127.0.0.1:7897 export HTTPS_PROXY=http://127.0.0.1:7897 export ALL_PROXY=http://127.0.0.1:7897

永久设置（推荐）：

# 编辑 zsh 配置文件 nano ~/.zshrc # 添加以下内容到文件末尾 export HTTP_PROXY=http://127.0.0.1:7897 export HTTPS_PROXY=http://127.0.0.1:7897 export ALL_PROXY=http://127.0.0.1:7897 # 保存并重新加载配置 source ~/.zshrc

创建代理开关函数（可选）：

# 在 ~/.zshrc 中添加 function proxy_on() { export HTTP_PROXY=http://127.0.0.1:7897 export HTTPS_PROXY=http://127.0.0.1:7897 export ALL_PROXY=http://127.0.0.1:7897 echo "代理已开启" } function proxy_off() { unset HTTP_PROXY unset HTTPS_PROXY unset ALL_PROXY echo "代理已关闭" } function proxy_status() { if [ -n "$HTTP_PROXY" ]; then echo "代理已开启: $HTTP_PROXY" else echo "代理未开启" fi }

3. 配置 API 密钥（如使用 API 计费）

如果选择使用 API 计费模式，需要设置 API 密钥：

# 临时设置 export ANTHROPIC_API_KEY=your-api-key-here # 永久设置 echo 'export ANTHROPIC_API_KEY=your-api-key-here' >> ~/.zshrc source ~/.zshrc
获取 API 密钥：
  1. 访问 Anthropic Console
  2. 登录您的账户
  3. 创建一个新的 API 密钥

4. 测试连接

验证网络连接和配置：

# 测试网络连接 ping api.anthropic.com # 测试 HTTPS 连接 curl -I https://api.anthropic.com # 测试 API 密钥（如果使用） curl -H "Authorization: Bearer $ANTHROPIC_API_KEY" \ -H "Content-Type: application/json" \ -d '{"model": "claude-3-sonnet-20240229", "max_tokens": 10, "messages": [{"role": "user", "content": "Hello"}]}' \ https://api.anthropic.com/v1/messages

🎯 首次启动和配置

1. 启动 Claude Code

claude-code

2. 选择显示主题

首次启动时，Claude Code 会让您选择终端主题：

  1. Dark mode（深色模式）
  2. Light mode（浅色模式）
  3. Dark mode (colorblind-friendly)（深色色盲友好模式）
  4. Light mode (colorblind-friendly)（浅色色盲友好模式）
  5. Dark mode (ANSI colors only)（深色 ANSI 颜色）
  6. Light mode (ANSI colors only)（浅色 ANSI 颜色）

输入数字 1-6 选择您喜欢的主题。

3. 选择计费方式

Claude Code 提供两种计费方式：

选项 1：Claude 订阅账户

  • 使用您的 Claude Pro/Team 订阅
  • 固定月费，使用量可预测
  • 适合：经常使用的用户

选项 2：Anthropic Console 账户

  • 按 API 使用量计费
  • 用多少付多少
  • 适合：偶尔使用或想控制成本的用户
💡 建议：
如果已有 Claude Pro 订阅：选择选项 1
如果只有 API 密钥：选择选项 2

4. API 密钥确认

如果系统检测到环境中的 API 密钥，会询问是否使用：

  • 如果选择了订阅计费：建议选择 "No"
  • 如果选择了 API 计费：选择 "Yes"

5. 文件夹权限确认

Claude Code 会询问是否信任当前工作目录，选择 "Yes, proceed" 以允许 Claude Code 读取和修改文件。

💻 基本使用

常用命令

命令 说明 示例
claude 启动交互式会话 -
claude "问题" 直接提问并启动会话 claude "如何使用Python读取CSV？"
claude -p "查询" 单次查询后退出 claude -p "解释这个错误"
claude -c 继续最近的对话 -
claude -r <ID> 恢复指定会话 claude -r session_12345
cat 文件 | claude -p "查询" 处理管道输入 cat error.log | claude -p "分析错误"
claude update 更新到最新版本 -

交互式命令（会话内）

命令 说明
/help 获取帮助
/clear 清除会话
/exit 退出会话
/status 查看当前设置状态
/theme 更改主题
/init 创建项目说明文件
/terminal-setup 设置终端集成

🔧 故障排除

1. 连接问题

错误： Failed to connect to api.anthropic.com

解决方案：

  • 检查网络连接
  • 确认代理设置
  • 验证 API 密钥
  • 检查地区支持（查看支持的国家列表）

2. 权限错误

错误： EACCES permission denied

解决方案：

  • 使用 sudo 安装
  • 修改 npm 全局目录权限
  • 使用 npx 运行

3. API 密钥问题

错误： HTTP/2 403

解决方案：

  • 检查 API 密钥是否正确
  • 确认账户状态
  • 验证 API 配额

4. 代理问题

错误： 代理连接失败

解决方案：

  • 确认 Clash Verge 正在运行
  • 检查代理端口设置
  • 验证代理环境变量
# 检查代理状态 proxy_status # 重新启用代理 proxy_on # 测试连接 curl -I https://api.anthropic.com

🔒 安全提醒

🔑 保护 API 密钥

不要在公共场所或聊天记录中暴露 API 密钥

🔄 定期更换密钥

建议定期在 Anthropic Console 中重新生成 API 密钥

📁 文件权限

只在信任的目录中运行 Claude Code

👁️ 代码审查

始终审查 Claude Code 生成的代码再执行

⚠️ 重要提醒：
  • Claude Code 可以读取和修改文件，请谨慎授权
  • 在生产环境中使用前，请充分测试生成的代码
  • 定期检查账单，避免意外费用

🌟 最佳实践

  1. 从小功能开始：先尝试简单的任务，然后逐步处理复杂项目
  2. 具体描述需求：像与工程师交流一样详细描述您的需求
  3. 验证建议：Claude Code 会提出修改建议，请仔细验证后再应用
  4. 增量开发：让 Claude 提出计划，然后逐步实现
  5. 保持会话上下文：在同一会话中保持话题连贯性，获得更好的帮助
  6. 善用命令参数：根据需要选择合适的命令参数提高效率
🚀 专业提示：
使用 claude -p 进行快速查询，使用 claude 进行深入讨论和迭代开发。
