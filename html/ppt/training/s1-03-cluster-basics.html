3. 智算集群基础概述
什么是智算集群
  • 专为 AI 训练/推理设计的大规模 GPU 计算集群
  • 核心目标：将数百~数千 GPU 组成统一算力池
  • 关键挑战：GPU 间高效通信 & 数据高速供给
  • 典型规模：千卡级 (训练) / 百卡级 (推理)
核心硬件组成
组件典型配置
GPU 服务器8×H100/H800 + 2×CPU
IB 交换机QM9700 NDR 400G
存储系统Lustre / GPFS 并行存储
管理网络25/100GbE 以太网
散热系统液冷 (冷板式/浸没式)
网络架构特点
  • 双网分离 — 计算网 (IB) + 管理/存储网 (Ethernet)
  • 无损网络 — IB 基于信用的流控，零丢包
  • RDMA — 绕过 CPU，GPU 直接读写远端内存
  • GPUDirect — GPU↔GPU / GPU↔存储 零拷贝
  • Fat-Tree — 非阻塞全带宽拓扑，等分带宽 1:1
与传统 HPC / 云计算对比
维度传统集群智算集群
核心算力CPU 为主GPU 为主
互联网络Ethernet/IBIB + NVLink
通信模式MPI 点对点集合通信为主
存储需求中等 I/O极高吞吐
功耗密度10-15kW/柜50-100kW/柜
典型训练工作流
数据加载 → 前向传播 → 反向传播 → 梯度同步 (AllReduce) → 参数更新 → Checkpoint 循环至收敛
