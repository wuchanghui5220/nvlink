13. 智算集群运维管理
运维三大支柱
  • 监控观测 — 实时感知集群健康
  • 自动化运维 — 减少人工干预
  • 风险管控 — 变更管理 + 应急响应
目标：128 台服务器 / 1024 GPU 的规模化运维
标准流程 (5 步)
  1. 需求分析与影响评估
  2. 方案设计与评审
  3. 测试验证 (灰度/金丝雀)
  4. 生产实施 (变更窗口)
  5. 监控验证 (回滚预案)
监控指标体系
  • GPU — 温度/利用率/显存/ECC
  • CPU/内存 — 负载/内存使用
  • 网络 — IB 端口状态/延迟/错误
  • 存储 — I/O 吞吐/延迟/容量
  • 作业 — 队列深度/等待时间
健康检查清单
  • 硬件扫描 — lspci / nvidia-smi
  • 网络连通 — ibstat / ibping
  • 存储完整性 — mount / df -h
  • 服务可用 — Slurm / OpenSM / UFM
  • 性能基线 — NCCL / ib_write_bw
作业调度 (Slurm)
  • sinfo — 查看集群分区与节点状态
  • squeue — 查看作业队列
  • sbatch job.sh — 提交批处理作业
  • srun --gres=gpu:8 — 交互式运行
  • scancel <jobid> — 取消作业
应急响应流程
  1. 发现 — 监控告警/用户报告
  2. 评估 — 影响范围与等级
  3. 处置 — 隔离故障节点/切换
  4. 恢复 — 根因分析 + 修复
