认识NCCL
NVIDIA集合通信库(NCCL)针对多GPU和多节点环境中的集合通信提供高性能、可扩展的原语。
核心原语包括`all-reduce`、`all-gather`、`reduce-scatter`、`broadcast`。
NCCL利用GPUDirect RDMA和NVLink等硬件加速技术实现低延迟高带宽通信。
# NCCL自动检测系统拓扑并选择最优通信算法
# 支持与PyTorch、TensorFlow等框架无缝集成
🔨
构建nccl-tests
`nccl-tests`是NVIDIA提供的开源基准测试工具，用于验证GPU间通信性能。
依赖MPI(Message Passing Interface)实现多节点通信，需预先安装OpenMPI。
git clone https://github.com/NVIDIA/nccl-tests.git
make MPI=1 MPI_HOME=/usr/local/mpi CUDA_HOME=/usr/local/cuda NCCL_HOME=/usr/local/nccl
📊
单节点测试
在8-GPU节点上运行`all_reduce_perf`测试，数据范围8B-128MB。
参数说明：-b(起始大小) -e(结束大小) -f(增长因子) -g(GPU数量)
# 单节点8GPU测试
./build/all_reduce_perf -b 8 -e 128M -f 2 -g 8
# 查看拓扑结构: nvidia-smi topo -m
🔧
硬件层面优化
**高性能GPU**: 使用A100、H100或L40S等具有高内存带宽的GPU。

**网络基础设施**: InfiniBand(200Gbps+)或高速以太网提供节点间通信。
# 启用GPUDirect RDMA: GPU直接通过网络通信，绕过CPU

多节点测试
8节点64GPU测试（每节点8GPU），需配置hostfile文件。
hostfile格式：192.168.1.101 slots=8，支持IP或主机名。
# hostfile示例配置
192.168.1.101 slots=8
# 多节点测试命令
mpirun -np 64 -N 8 --hostfile hostfile --allow-run-as-root -x LD_LIBRARY_PATH ./build/all_reduce_perf -b 8 -e 8G -f 2 -g 1
📈
解释测试结果
输出显示操作平均时间和两种带宽：算法带宽(algorithm)和总线带宽(bus)。
**Bus Bandwidth**是关键指标，反映GPU间通信硬件使用效率。
# 单机内部: 接近NVLink理论带宽(600GB/s+)
# 多节点: 接近网络带宽(InfiniBand 200Gbps+)
# Bus Bandwidth通过公式转换，便于不同GPU数量性能比较
⚙️
环境变量调优
**NCCL_ALGO**: `Tree`(大规模)、`Ring`(小消息)或`CollNet`。
**NCCL_PROTO**: `Simple`(高带宽)、`LL`(低延迟)或`LL128`。
**NCCL_NTHREADS**: 每GPU线程数，**NCCL_BUFFSIZE**: 通信缓冲区大小。
export NCCL_DEBUG=INFO # 启用调试日志
export NCCL_TOPO_FILE=/path/to/topo.xml # 拓扑感知
# 通常无需手动设置，NCCL自动选择最优参数

AI训练网络优化
**批量大小优化**: 增大batch size减少通信频率，提高NCCL效率。
**梯度累积**: 使用gradient accumulation和all-reduce优化减少通信开销。
**混合并行**: 数据并行+模型并行混合策略平衡计算和通信。
