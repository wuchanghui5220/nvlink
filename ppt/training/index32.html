UFM安装操作
UFM软件安装操作文档
Docker容器化安装 · 高可用部署 · 配置管理
📋
预部署要求
1
安装基础软件包
在两台服务器上安装pacemaker、pcs和drbd-utils
U
Ubuntu系统
apt install pcs pacemaker drbd-utils
C
CentOS/Red Hat
yum install pcs pacemaker drbd84-utils kmod-drbd84
2
准备DRBD分区
在每台服务器上创建同名分区(如/dev/sdb1)，推荐大小10-20GB
3
配置主机名解析, 并配置免认证登录
确保hostname -i命令返回正确的管理接口IP，更新/etc/hosts文件，2台主机root用户免认证登录
4
创建UFM目录
在每台服务器上创建/opt/ufm/files/目录并设置读写权限
mkdir -p /opt/ufm/files/
chmod 755 /opt/ufm/files/
🐳
UFM容器安装
1
主服务器安装
在主服务器上安装UFM Enterprise容器
docker run -it --name=ufm_installer --rm \
-v /var/run/docker.sock:/var/run/docker.sock \
-v /etc/systemd/system/:/etc/systemd_files/ \
-v /opt/ufm/files/:/installation/ufm_files/ \
-v /tmp/license_file/:/installation/ufm_licenses/ \
mellanox/ufm-enterprise:latest \
--install
2
备用服务器安装
在备用服务器上安装UFM Enterprise容器
docker run -it --name=ufm_installer --rm \
-v /var/run/docker.sock:/var/run/docker.sock \
-v /etc/systemd/system/:/etc/systemd_files/ \
-v /opt/ufm/files/:/installation/ufm_files/ \
mellanox/ufm-enterprise:latest \
--install
🔄
UFM高可用安装
1
下载HA软件包
在两台服务器上下载UFM-HA包
wget https://www.mellanox.com/downloads/ufm/ufm_ha/5.8.0/ufm_ha_5.8.0-4.tgz
wget https://www.mellanox.com/downloads/ufm/ufm_ha/5.8.0/ufm_ha_5.8.0-4.sha256
2
解压并安装
在两台服务器上解压到/tmp/并运行安装脚本
cd /tmp/ufm_ha_XXX
./install.sh -l /opt/ufm/files/ -d /dev/sdb1 -p enterprise
⚙️
配置与启动
1
配置HA集群
在主服务器上配置双链路HA节点
configure_ha_nodes.sh \
--cluster-password 12345678 \
--master-primary-ip 10.10.50.1 \
--standby-primary-ip 10.10.50.2 \
--master-secondary-ip 192.168.10.1 \
--standby-secondary-ip 192.168.10.2 \
--virtual-ip 10.10.50.3
2
查看DRBD同步状态
使用drbdadm命令，查看DRBD同步状态，等待同步完成。同步完成状态如下：
[root@ufm-01 ~]# drbdadm status
ha_data role:Primary
  disk:UpToDate
  peer role:Secondary
    replication:Established peer-disk:UpToDate

[root@ufm-02 ~]# drbdadm status
ha_data role:Secondary
  disk:UpToDate
  peer role:Primary
    replication:Established peer-disk:UpToDate
3
启动和管理
UFM HA集群管理命令
# 启动集群
ufm_ha_cluster start
# 查看状态
ufm_ha_cluster status
# 停止集群
ufm_ha_cluster stop
⚠️
重要提示
1. 安装前需关闭防火墙或开放必要端口
2. 需要禁用SELinux
3. 等待DRBD同步完成后再启动集群
4. 卸载时先停止集群再运行/opt/ufm/ufm_ha/uninstall_ha.sh
