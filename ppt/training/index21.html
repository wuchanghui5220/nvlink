Docker容器化和GPU服务管理配置
容器化工具配置
🐳
安装Docker
安装Docker容器运行时环境，为后续NVIDIA Container Toolkit安装做准备。
sudo apt update && sudo apt install docker.io -y
🔧
NVIDIA Container Toolkit
下载GPG密钥并添加NVIDIA容器工具包的软件仓库。
这是在Docker容器中使用GPU的必要组件。
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
⚙️
添加软件仓库并安装
添加NVIDIA容器工具包的软件仓库，然后更新包列表并安装。
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
服务管理与优化
🔄
配置Docker运行时
配置Docker以使用NVIDIA运行时，并重启Docker服务, Docker就可以识别和使用GPU资源了。
sudo nvidia-ctk runtime configure --runtime=docker
🔗
NVIDIA Fabric Manager
对于NVLink连接的多GPU系统，需要安装Fabric Manager管理GPU间的通信，并启动服务。
下载页面: NVIDIA Fabric Manager
dpkg -i cuda-drivers-fabricmanager-575_575.57.08-1_amd64.deb
sudo systemctl enable nvidia-fabricmanager
🔄
开启服务
启用NVIDIA GPU持久化服务
sudo systemctl enable nvidia-persistenced
⚡
可选配置
设置功耗限制: 例如限制为600W以降低热量。
启用纠错码(ECC): 提高数据完整性，适用于科学计算。
sudo nvidia-smi -pl 600
sudo nvidia-smi -e 1
⚙️
配置要点：安装Docker和NVIDIA Container Toolkit，启用持久化服务，根据需要配置功耗限制和ECC
